Game of Life

Reactive implementation of Conway's Game of Life. Users can generate a seed configuration automatically by using a the populate button which assigns alive cells randomly, or create their own seed by clicking at a cell. Each new generation is computed following these rules:

  1. Any live cell with fewer than two live neighbours dies, as if caused by under-population.
  2. Any live cell with two or three live neighbours lives on to the next generation.
  3. Any live cell with more than three live neighbours dies, as if by over-population.
  4. Any dead cell with exactly three live neighbours becomes a live cell, as if by reproduction.

The implementation does not use any component state. Instead, all data is stored in a central ngrx-store and components are reactive, i.e., their only input are RxJS observables. The source code can be found at Github.
